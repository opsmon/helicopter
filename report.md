# 6. Поддержка GitHub Pages

Проект должен корректно работать:

* на пользовательском домене GitHub Pages;
* в поддиректории репозитория;
* с кастомным доменом;
* при прямом открытии внутренних страниц;
* после обновления страницы;
* с относительными путями к изображениям;
* с относительными путями к статическим ресурсам.

Если сайт публикуется по адресу:

```text
https://username.github.io/granat-demo/
```

необходимо учитывать `basePath` и `assetPrefix`.

Пример:

```ts
const isGitHubPages = process.env.GITHUB_ACTIONS === "true";
const repositoryName = "granat-demo";

const nextConfig = {
  output: "export",
  trailingSlash: true,

  basePath: isGitHubPages ? `/${repositoryName}` : "",
  assetPrefix: isGitHubPages ? `/${repositoryName}/` : "",

  images: {
    unoptimized: true,
  },
};
```

Название репозитория вынести в отдельную переменную или environment variable.

Не хардкодить путь репозитория во всех компонентах.

  details: Record<string, string | number | boolean>;
}

# 52. GitHub Actions для GitHub Pages

Создать workflow:

```text
.github/workflows/deploy.yml
```

Использовать официальный механизм GitHub Pages.

Пример workflow:

```yaml
name: Deploy to GitHub Pages

on:
  push:
    branches:
      - main

  workflow_dispatch:

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: pages
  cancel-in-progress: true

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: npm

      - name: Install dependencies
        run: npm ci

      - name: Lint
        run: npm run lint

      - name: Typecheck
        run: npm run typecheck

      - name: Build
        run: npm run build
        env:
          GITHUB_ACTIONS: true

      - name: Add nojekyll
        run: touch out/.nojekyll

      - name: Upload artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: ./out

  deploy:
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}

    runs-on: ubuntu-latest
    needs: build

    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
```

Workflow должен:

* запускаться при push в `main`;
* запускаться вручную;
* устанавливать зависимости;
* запускать lint;
* запускать typecheck;
* собирать проект;
* публиковать директорию `out`;
* создавать `.nojekyll`.
